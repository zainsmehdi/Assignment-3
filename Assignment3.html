Task 3

Name Web Attacks
Malware
Malware is a term for various pieces of software that are intentionally designed to damage a computer, server, client or computer network. Malware often operates covertly, exploiting vulnerabilities in a system's security to gain unauthorized access or manipulate data.
It can be distributed through various means, including email attachments, infected websites, removable storage devices, and malicious downloads.
McAfee
XSS Attacks
Cross-Site Scripting (XSS) attacks, where malicious scripts are injected into trusted websites through vulnerable web applications.
Attackers exploit these vulnerabilities to send harmful code to unsuspecting users' browsers, allowing them to access sensitive information like cookies and session tokens.
XSS attacks pose significant risks by deceiving browsers into executing scripts from seemingly trusted sources, ultimately compromising the security of web users.
owasp.org
Man in the Middle (MITM) attacks A man-in-the-middle (MITM) attack occurs when a perpetrator intercepts communication between a user and an application to eavesdrop or impersonate one party.
The aim is to steal personal information like login credentials and credit card details, often targeting users of financial apps, e-commerce sites, and other platforms requiring authentication.
This stolen information can be used for identity theft, unauthorized transactions, or as part of advanced persistent threat (APT) infiltration tactics.
www.imperva.com
SQL Injection
A SQL injection attack involves inserting SQL queries into client input data, enabling unauthorized access to or manipulation of a database.
Successful exploitation can lead to the extraction of sensitive data, alteration of database contents, execution of administrative tasks, and even issuing commands to the operating system.
These attacks exploit vulnerabilities in data input mechanisms, allowing attackers to control SQL commands executed by the application.
owasp.org
Ransomware
Ransomware is a common and dangerous type of malware.
It works by locking up or encrypting your files so you can no longer access them.
A ransom, usually in the form of cryptocurrency, is demanded to restore access to the files. Cybercriminals might also demand a ransom to prevent data and intellectual property from being leaked or sold online.
Cyber.gov.au
DOS Attacks
A denial-of-service (DoS) attack is a type of cyber attack in which a malicious actor aims to render a computer or other device unavailable to its intended users by interrupting the device's normal functioning.
DoS attacks typically function by overwhelming or flooding a targeted machine with requests until normal traffic is unable to be processed, resulting in denial-of-service to addition users.
A DoS attack is characterized by using a single computer to launch the attack.
cloudflare.com.au
Phishing
Phishing is a way cyber criminals trick you into giving them personal information.
They send you fraudulent emails or text messages often pretending to be from large organisations you know or trust.
They may try to steal your online banking logins, credit card details or passwords. Phishing can result in the loss of information, money or identity theft. 
Spear-phishing is when these emails and text messages are highly targeted to the recipient.
Cyber.gov.au
DNS Spoofing
DNS spoofing attacks involve manipulating DNS resolution to redirect users to malicious websites by altering DNS records or intercepting DNS queries.
Attackers may use DNS spoofing to impersonate legitimate websites, leading users to unknowingly divulge sensitive information such as login credentials or financial details.
Mitigation strategies against DNS spoofing include implementing DNSSEC (Domain Name System Security Extensions), using reputable DNS servers, and monitoring DNS traffic for signs of tampering or unauthorized redirection.
imperva.com
Clickjacking
Clickjacking involves concealing malicious actions or links beneath seemingly harmless elements on a webpage, manipulating users into clicking on them without their knowledge.
By overlaying transparent or opaque layers on top of legitimate content, attackers can trick users into performing actions such as clicking on ads, sharing content, or unknowingly activating browser features.
Countermeasures against clickjacking include employing frame-busting scripts, implementing X-Frame-Options headers, and ensuring strict content security policies to prevent malicious embedding of website content.
owasp.org
Cross-Site Request Forgery (CSRF)
CSRF attacks exploit the trust between a user and a website, tricking the user into unintentionally executing actions they did not intend to perform.
Attackers may craft malicious links or forms that, when clicked or submitted by authenticated users, perform actions such as transferring funds, changing account settings, or deleting data.
To mitigate CSRF attacks, web applications often employ techniques such as anti-CSRF tokens, which verify the authenticity of requests, ensuring they originated from the legitimate user session.
owasp.org
